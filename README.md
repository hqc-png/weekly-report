# Weekly Report Generator

AI-powered weekly report generator that fetches your GitHub commits and creates professional weekly reports using Claude AI.

## Features

- 🔐 GitHub OAuth authentication
- 📊 Fetch commits from multiple repositories
- 🤖 AI-powered summarization with Claude
- 📝 Generate beautiful Markdown reports
- 💾 Save and manage historical reports
- 🚀 **CLI mode**: Generate reports without browser (one command!)
- 📱 Responsive design for mobile and desktop

## Tech Stack

- **Framework**: Next.js 16.2 with App Router
- **Language**: TypeScript
- **Styling**: Tailwind CSS 4
- **Authentication**: NextAuth.js v5 with GitHub OAuth
- **GitHub API**: Octokit
- **AI**: Anthropic Claude API (via SDK)
- **Date utilities**: date-fns
- **Markdown**: react-markdown + remark-gfm

## Getting Started

### Web Application

#### 1. Environment Setup

Create `.env.local` file:

```bash
# NextAuth.js
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=<generate with: openssl rand -base64 32>

# GitHub OAuth (create at: https://github.com/settings/developers)
GITHUB_CLIENT_ID=<your_client_id>
GITHUB_CLIENT_SECRET=<your_client_secret>

# Anthropic Claude API
ANTHROPIC_API_KEY=<your_api_key>
ANTHROPIC_BASE_URL=https://api.anthropic.com  # Optional, for custom endpoint
ANTHROPIC_MODEL=claude-3-5-sonnet-20241022     # Optional, default model
```

#### 2. Installation

```bash
npm install
```

#### 3. Development

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and sign in with GitHub.

#### 4. Build

```bash
npm run build
npm start
```

### CLI Mode (Headless Report Generation)

Generate weekly reports without opening a browser - perfect for automation!

#### 1. First-time Setup

```bash
npm run report:init
```

This will prompt you to enter:
- GitHub Personal Access Token (create at: https://github.com/settings/tokens)
  - Required scope: `repo` (for accessing private repositories)
- GitHub username
- Anthropic API key
- Default repository list (comma-separated, e.g., `owner/repo1,owner/repo2`)
- Output directory for reports (default: `~/Desktop/WeeklyReports`)

Configuration is saved to `~/.weekly-report/config.json`

#### 2. Generate Reports

```bash
# Generate current week report (Monday to Sunday)
npm run report

# Generate last week's report
npm run report:last-week

# Custom date range
npx tsx scripts/generate-report.ts --since 2026-04-01 --until 2026-04-07
```

#### 3. Claude Code Skill

If you're using Claude Code, you can invoke the skill:

```
/generate-report              # Current week
/generate-report --week last  # Last week
```

The report will be saved as a Markdown file in your configured output directory.

## Configuration

### Web App Config (`.env.local`)

See "Getting Started > Web Application" section above.

### CLI Config (`~/.weekly-report/config.json`)

Generated by `npm run report:init`. Format:

```json
{
  "github": {
    "token": "ghp_xxxxxxxxxxxx",
    "username": "your-username"
  },
  "anthropic": {
    "apiKey": "sk-ant-xxxxxxxxxxxx",
    "model": "anthropic--claude-4.5-sonnet"  // Optional
  },
  "repositories": [
    "owner/repo1",
    "owner/repo2"
  ],
  "outputDirectory": "C:\\Users\\YourName\\Desktop\\WeeklyReports"
}
```

## How It Works

### Web App Flow

1. User signs in with GitHub OAuth
2. Select repositories from dropdown (fetched via GitHub API)
3. Choose date range (defaults to current week)
4. Click "Generate Report"
5. Backend fetches commits filtered by authenticated user
6. Claude API summarizes commits into structured report
7. Report displayed in browser with Markdown rendering
8. Download or view historical reports

### CLI Flow

1. Load config from `~/.weekly-report/config.json`
2. Calculate date range (current week, last week, or custom)
3. Fetch commits using GitHub Personal Access Token
4. Filter commits by configured username
5. Send to Claude API for summarization
6. Save Markdown file to configured output directory
7. Display success message with file path

## Common Issues

### "No commits found"

- Check that repositories are accessible with your token/OAuth
- Verify date range includes actual commit dates
- Ensure commits are authored by the configured username

### GitHub API Rate Limits

- Authenticated requests: 5000/hour
- Use Personal Access Token for higher limits
- Add delays between multiple report generations

### Claude API Errors

- Verify API key is valid
- Check network connectivity
- Ensure API endpoint is correct (for custom base URL)
