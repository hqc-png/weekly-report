# 周报工具需求规格说明书

## 1. 项目概述

一个个人使用的周报工具，支持每天随时记录完成的任务，并在周末自动汇总生成 Markdown 格式的周报。

## 2. 目标用户

- **主要用户**: 个人使用
- **使用场景**: 日常工作任务记录与周报生成

## 3. 核心功能需求

### 3.1 任务管理
- 支持每天随时添加完成的任务
- 任务需要关联到具体项目
- 任务包含基本信息：任务描述、完成时间、所属项目

### 3.2 项目管理
- 项目可以动态添加
- 添加任务时可以选择现有项目或创建新项目
- 支持"其他"或"杂项"分类（可选，用于未分类任务）

### 3.3 周报生成
- 周末自动汇总本周任务
- 生成 Markdown 格式的周报文件
- 周报按项目分类展示任务
- 周报文件自动保存到指定目录

### 3.4 历史管理
- 生成的周报文件按日期命名并保存
- 保留历史周报记录，可供后续查看

## 4. 用户故事

### Story 1: 添加任务
```
作为用户
我想要随时添加今天完成的任务
以便及时记录工作进展
```
**验收标准**:
- 打开 GUI 界面
- 输入任务描述
- 选择或创建项目
- 任务成功保存并显示确认

### Story 2: 创建新项目
```
作为用户
我想要在添加任务时创建新项目
以便灵活组织不同类型的工作
```
**验收标准**:
- 在添加任务时可以输入新项目名称
- 新项目自动添加到项目列表
- 后续可以继续使用该项目

### Story 3: 查看本周任务
```
作为用户
我想要查看本周已记录的任务
以便了解当前工作进展
```
**验收标准**:
- 可以打开界面查看本周任务列表
- 任务按项目分组显示
- 显示任务的完成时间

### Story 4: 生成周报
```
作为用户
我想要在周末自动生成周报
以便快速整理一周的工作成果
```
**验收标准**:
- 周末自动触发周报生成（或手动触发）
- 生成的周报为 Markdown 格式
- 周报按项目分类展示任务
- 周报文件保存到指定目录

## 5. 技术决策

### 5.1 技术栈
- **编程语言**: Python（推荐，适合快速开发 GUI）
- **GUI 框架**: Tkinter（Python 内置，无需额外安装）或 PyQt（功能更强大）
- **数据存储**: JSON 文件（简单轻量）
- **周报格式**: Markdown

### 5.2 运行环境
- **操作系统**: Windows
- **Python 版本**: 3.8+

### 5.3 数据结构设计

#### 任务数据结构
```json
{
  "id": "unique_id",
  "description": "任务描述",
  "project": "项目名称",
  "completed_at": "2026-04-09T10:30:00",
  "week": "2026-W15"
}
```

#### 项目数据结构
```json
{
  "projects": [
    "项目A",
    "项目B",
    "其他"
  ]
}
```

### 5.4 文件组织
```
weekly-report/
├── data/
│   ├── tasks.json          # 任务数据
│   └── projects.json       # 项目列表
├── reports/
│   ├── 2026-W14.md        # 历史周报
│   └── 2026-W15.md        # 当前周报
├── src/
│   ├── main.py            # 主程序入口
│   ├── gui.py             # GUI 界面
│   ├── task_manager.py    # 任务管理逻辑
│   └── report_generator.py # 周报生成逻辑
└── SPEC.md                # 本文档
```

### 5.5 周报格式示例
```markdown
# 第 15 周周报 (2026-04-07 至 2026-04-13)

## 项目A
- 完成功能模块开发 (2026-04-09)
- 修复登录bug (2026-04-10)
- 代码审查 (2026-04-11)

## 项目B
- 需求分析会议 (2026-04-08)
- 编写技术方案 (2026-04-12)

## 其他
- 学习新技术 (2026-04-09)

---
**总计**: 6 个任务
```

## 6. 功能优先级

### P0 (必须有)
- 添加任务功能
- 选择/创建项目
- 生成 Markdown 周报
- 数据持久化存储

### P1 (应该有)
- 查看本周任务列表
- 手动触发周报生成
- 任务统计（任务数量）

### P2 (可以有)
- 编辑/删除任务
- 任务搜索
- 周报模板自定义
- 自动周报提醒

## 7. 非功能性需求

### 7.1 性能要求
- GUI 响应时间 < 1秒
- 周报生成时间 < 3秒

### 7.2 可用性要求
- 界面简洁直观
- 操作步骤少于 3 步
- 提供操作反馈

### 7.3 可靠性要求
- 数据自动保存
- 异常情况不丢失数据
- 生成周报前备份数据

## 8. 未来扩展可能

- 数据导出功能（Excel、PDF）
